# FEATURES.md

## 문서 목적

이 문서는 `GOAL.md`와 `PRODUCT.md`를 바탕으로 PRZM의 기능 범위를 정의한다.

목적은 단순 아이디어 목록을 만드는 것이 아니라, 무엇이 MVP에 반드시 포함되어야 하는지와 무엇이 출시 이후로 미뤄질 수 있는지를 명확히 하는 것이다.

초기 PRZM의 MVP는 아래 질문에 실제로 답할 수 있어야 한다.

> 우리 동네에서 지금 어떤 질병이 우리 아이 연령대에 퍼지고 있나?

그리고 그 답을 현재 상태에 그치지 않고, 앞으로 일주일과 한달의 확산 및 소멸 방향까지 이어서 보여줄 수 있어야 한다.

## MVP 정의

PRZM의 MVP는 다음 경험이 한 흐름으로 완성된 상태를 의미한다.

1. 사용자가 지도에서 자신의 지역을 찾는다.
2. 사용자가 어린이 연령을 선택한다.
3. 사용자가 현재 지역 위험도와 확산 범위를 확인한다.
4. 사용자가 특정 질병을 선택해 상세 분석을 본다.
5. 사용자가 앞으로 일주일, 한달의 확산 또는 소멸 방향을 확인한다.

즉, 지도 탐색, 상세 분석, 단기 예측이 모두 연결되어야 MVP로 본다.

## MVP 핵심 기능

### 1. 지도 기반 지역 위험도 표시

설명:
사용자는 첫 화면에서 자신의 지역 위험도를 지도 위에서 바로 확인할 수 있어야 한다.

사용자 가치:
복잡한 표를 읽지 않고도 내 주변 상황을 빠르게 이해할 수 있다.

출시 필수 여부:
필수

포함 요소:

- 서울특별시/경기도 범위 지도
- 지역 단위 위험도 시각화
- 현재 위험이 높은 지역 강조

### 2. 질병별 확산 영역 표시

설명:
사용자는 특정 질병이 어느 지역에 퍼지고 있는지 지도 위에서 확인할 수 있어야 한다.

사용자 가치:
위험이 단순히 높다/낮다가 아니라 실제로 어디에 퍼지고 있는지 알 수 있다.

출시 필수 여부:
필수

포함 요소:

- 질병 선택
- 선택 질병의 확산 범위 표시
- 지역별 분포 차이 확인

### 3. 연령 필터링

설명:
사용자는 아이의 나이에 맞춰 데이터를 다시 볼 수 있어야 한다.

사용자 가치:
전체 유행 정보가 아니라 내 아이에게 가까운 위험을 볼 수 있다.

출시 필수 여부:
필수

포함 요소:

- 0세부터 13세까지 1세 단위 선택
- 선택 연령에 따라 지도와 요약 정보 갱신

### 4. 지역 선택 및 탐색

설명:
사용자는 현재 위치와 무관하게 관심 지역을 직접 찾아볼 수 있어야 한다.

사용자 가치:
거주지, 학교 주변, 가족이 사는 지역 등 다양한 기준으로 확인할 수 있다.

출시 필수 여부:
필수

포함 요소:

- 지도 이동과 확대/축소
- 특정 지역 선택
- 선택 지역 상태 유지

### 5. 요약 정보 패널

설명:
지도만으로 놓칠 수 있는 핵심 내용을 짧은 문장과 요약 지표로 보여준다.

사용자 가치:
사용자는 데이터를 해석하지 않아도 지금 중요한 내용을 빠르게 이해할 수 있다.

출시 필수 여부:
필수

포함 요소:

- 현재 주의가 필요한 질병
- 최근 변화 방향
- 선택 지역과 연령 기준 요약

### 6. 상세 분석 탭

설명:
사용자는 특정 질병 또는 지역을 더 깊게 분석할 수 있어야 한다.

사용자 가치:
현재 위험뿐 아니라 왜 위험한지와 어떻게 변하는지를 볼 수 있다.

출시 필수 여부:
필수

포함 요소:

- 기간별 추세
- 확산 속도 변화
- 연령별 분포
- 성별 분포

### 7. 일주일 예측

설명:
사용자는 앞으로 일주일 동안 위험이 증가하는지 감소하는지 볼 수 있어야 한다.

사용자 가치:
가까운 미래의 변화에 대비할 수 있다.

출시 필수 여부:
필수

포함 요소:

- 선택 질병/지역 기준 일주일 예측
- 증가/정체/감소 방향 제시

### 8. 한달 예측

설명:
사용자는 단기 흐름을 넘어 한달 수준의 확산 또는 소멸 가능성을 볼 수 있어야 한다.

사용자 가치:
지속적인 위험인지, 일시적인 확산인지 구분할 수 있다.

출시 필수 여부:
필수

포함 요소:

- 선택 질병/지역 기준 한달 예측
- 확산 또는 소멸 방향 해석

## MVP 지원 기능

아래 기능은 MVP의 핵심 흐름을 성립시키기 위해 필요하지만, 사용자가 직접 "대표 기능"으로 인식하지 않을 수도 있다.

### 1. 기본 필터 상태

- 초기 진입 시 기본 지역 또는 추천 지역 설정
- 기본 연령 상태 설정
- 질병 미선택 상태에서의 기본 요약 제공

### 2. 데이터 갱신 시점 표시

- 현재 데이터가 언제 기준인지 표시
- 예측이 어떤 시점을 기준으로 계산되었는지 표시

### 3. 빈 상태 처리

- 선택한 조건에 데이터가 부족할 때의 안내
- 특정 질병 또는 연령에서 의미 있는 데이터가 없을 때의 안내

### 4. 모바일 대응

- 모바일에서 지도와 패널이 무너지지 않도록 구성
- 작은 화면에서도 지역 탐색과 연령 선택이 가능해야 함

### 5. 용어 단순화

- 일반 시민이 이해할 수 있는 표현 사용
- 지나치게 전문적인 역학 용어는 피하고 해석 중심 문장 제공

## 출시 이후 보강 기능

아래 기능은 제품 가치를 높일 수 있지만, MVP 성립의 절대 조건은 아니다.

### 1. 관심 지역 저장

- 자주 보는 지역 저장
- 다음 방문 시 빠르게 재확인

### 2. 비교 보기

- 두 지역 비교
- 두 연령대 비교
- 두 질병 비교

### 3. 알림 기능

- 특정 지역 위험 상승 알림
- 특정 연령대 위험 변화 알림

### 4. 해석 보강 콘텐츠

- 왜 위험도가 높아졌는지 설명
- 예측 결과를 읽는 방법 안내

### 5. 시간 슬라이더

- 과거 며칠 또는 몇 주의 변화 흐름 재생

## 확장 기능

초기 운영 범위를 넘어서는 기능은 아래와 같다.

### 1. 전국 확장

- 서울/경기도 외 지역 지원
- 전국 단위 탐색 경험

### 2. 전 연령 확장

- 어린이 외 청소년, 성인, 고령층 포함

### 3. 고급 인구학 분석

- 성별 외 다양한 특성 기준 분석
- 복합 조건 필터링

### 4. 전문가용 분석 기능

- 행정/보건 담당자를 위한 고급 데이터 보기
- 더 상세한 추세 해석과 데이터 다운로드

## 기능 우선순위 원칙

- 지도에서 현재 위험을 이해하는 기능이 가장 우선이다.
- 상세 분석은 지도 경험을 설명하고 확장하는 역할이어야 한다.
- 예측은 MVP에 포함되지만, 복잡성보다 해석 가능성이 우선이다.
- 초기 범위에서는 서울/경기도와 0~13세 문제를 깊게 푸는 것이 중요하다.
- 기능 추가보다 핵심 질문에 더 빠르고 명확하게 답하는 것이 중요하다.

## 출시 체크 기준

PRZM MVP는 아래 기준을 만족할 때 출시 가능하다고 본다.

- 사용자가 지도에서 서울/경기도 내 지역 위험도를 확인할 수 있다.
- 사용자가 0세부터 13세까지의 연령을 선택해 결과를 다시 볼 수 있다.
- 사용자가 특정 질병의 확산 범위를 확인할 수 있다.
- 사용자가 상세 분석 탭에서 추세와 분포를 볼 수 있다.
- 사용자가 앞으로 일주일 및 한달의 확산/소멸 방향을 확인할 수 있다.
- 사용자가 복잡한 설명 없이도 현재 위험과 가까운 미래를 이해할 수 있다.

이 문서는 이후 `IA`, `USER FLOWS`, `DATA`, `PREDICTION`, `UI` 문서에서 더 세부적으로 분해한다.
