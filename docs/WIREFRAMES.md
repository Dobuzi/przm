# WIREFRAMES.md

## 문서 목적

이 문서는 PRZM의 MVP 화면을 저해상도 와이어프레임 관점에서 정리한다.

목적은 시각 디자인을 확정하는 것이 아니라, 어떤 정보가 어디에 놓이고 어떤 상태에서 어떻게 열리는지를 배치 수준에서 고정하는 것이다.

PRZM은 단일 지도 화면 중심 제품이므로, 와이어프레임 역시 "여러 페이지"보다 "한 화면 안의 상태 변화"를 설명하는 방식으로 작성한다.

## 와이어프레임 원칙

- 지도는 항상 가장 큰 영역을 차지한다.
- 기본 상태에서는 요약 정보가 가볍게 보이고, 상세 상태에서는 분석 정보가 단계적으로 열린다.
- 데스크톱은 `지도 위 요약 카드 → 오른쪽 상세 패널` 구조를 따른다.
- 모바일은 `하단 요약 카드 → 확장 드로어` 구조를 따른다.
- 현재 상태, 분석, 예측의 정보 계층은 항상 유지한다.
- 사용자는 언제나 현재 선택한 `지역 / 연령 / 질병` 상태를 확인할 수 있어야 한다.

## 데스크톱 상태별 와이어프레임

## D1. 초기 진입 상태

### 목적

사용자가 진입 직후 현재 위험 신호를 빠르게 훑어보는 상태다.

### 구조 설명

- 상단에는 짧은 제어 바가 있다.
- 화면 대부분은 지도다.
- 지도 위에는 현재 핵심 신호를 요약하는 카드가 하나 또는 소수로 뜬다.
- 아직 오른쪽 상세 패널은 열리지 않는다.

```text
+----------------------------------------------------------------------------------+
| PRZM | 지역 선택 | 질병 선택 | 연령 선택 | 기준 시점                             |
+----------------------------------------------------------------------------------+
|                                                                                  |
|   +-----------------------------------+                                          |
|   | 현재 주의 질병                    |                                          |
|   | 서울 서남권 위험 높음             |                                          |
|   | 최근 확산세 커지는 중             |                                          |
|   +-----------------------------------+                                          |
|                                                                                  |
|                              MAIN MAP                                            |
|                     (서울 / 경기 위험도 색상 표현)                               |
|                                                                                  |
|                                                                                  |
|                                                                                  |
|                                                                                  |
+----------------------------------------------------------------------------------+
```

### 핵심 포인트

- 첫 화면은 보고서가 아니라 지도 서비스처럼 보여야 한다.
- 요약 카드는 짧고 강해야 한다.
- 사용자는 지도만 봐도 위험 지역을 대략 파악할 수 있어야 한다.

## D2. 지역 선택 상태

### 목적

사용자가 특정 지역을 선택해 "우리 동네" 기준으로 좁혀 보는 상태다.

### 구조 설명

- 상단 제어 바는 유지된다.
- 지도에서는 선택 지역이 분명히 강조된다.
- 요약 카드는 해당 지역 기준으로 갱신된다.

```text
+----------------------------------------------------------------------------------+
| PRZM | 지역: 성남시 분당구 | 질병 선택 | 연령 선택 | 기준 시점                    |
+----------------------------------------------------------------------------------+
|                                                                                  |
|   +-----------------------------------+                                          |
|   | 분당구 현재 위험도: 높음          |                                          |
|   | 주의 질병: A 질병                 |                                          |
|   | 최근 7일 확산세 증가              |                                          |
|   +-----------------------------------+                                          |
|                                                                                  |
|                              MAIN MAP                                            |
|                      [선택 지역 하이라이트 / 주변 지역 비교]                     |
|                                                                                  |
|                                                                                  |
|                                                                                  |
|                                                                                  |
+----------------------------------------------------------------------------------+
```

### 핵심 포인트

- 선택 지역은 시각적으로 즉시 구분되어야 한다.
- 요약은 전체 지역 설명이 아니라 선택 지역 해석으로 바뀌어야 한다.

## D3. 연령 필터 적용 상태

### 목적

사용자가 특정 연령 기준으로 위험을 다시 읽는 상태다.

### 구조 설명

- 상단 제어 바에 연령 선택 결과가 명확히 표시된다.
- 지도와 요약 카드가 연령 기준으로 다시 계산된 결과를 보여준다.

```text
+----------------------------------------------------------------------------------+
| PRZM | 지역: 분당구 | 질병 선택 | 연령: 7세 | 기준 시점                           |
+----------------------------------------------------------------------------------+
|                                                                                  |
|   +-----------------------------------+                                          |
|   | 7세 기준 현재 위험도: 높음        |                                          |
|   | A 질병 비중이 높게 나타남         |                                          |
|   | 전체 어린이 평균보다 위험 높음    |                                          |
|   +-----------------------------------+                                          |
|                                                                                  |
|                              MAIN MAP                                            |
|                   [7세 기준 위험도 / 확산 범위 반영된 지도]                      |
|                                                                                  |
|                                                                                  |
|                                                                                  |
|                                                                                  |
+----------------------------------------------------------------------------------+
```

### 핵심 포인트

- 사용자는 지금 전체가 아니라 특정 연령 기준으로 보고 있다는 사실을 잊지 않아야 한다.
- 연령 선택은 결과 변화를 즉시 체감하게 해야 한다.

## D4. 질병 선택 상태

### 목적

사용자가 특정 질병 중심으로 확산 범위를 읽는 상태다.

### 구조 설명

- 상단 제어 바에 질병명이 표시된다.
- 지도는 해당 질병의 확산 범위를 강조한다.
- 요약 카드는 질병 중심 메시지로 바뀐다.

```text
+----------------------------------------------------------------------------------+
| PRZM | 지역: 분당구 | 질병: A 질병 | 연령: 7세 | 기준 시점                        |
+----------------------------------------------------------------------------------+
|                                                                                  |
|   +-----------------------------------+                                          |
|   | A 질병 현재 위험도: 높음          |                                          |
|   | 분당구 및 인접 지역 확산 중       |                                          |
|   | 상세 분석 보기 >                  |                                          |
|   +-----------------------------------+                                          |
|                                                                                  |
|                              MAIN MAP                                            |
|                    [A 질병 확산 범위 / 위험도 색상 단계 강조]                    |
|                                                                                  |
|                                                                                  |
|                                                                                  |
|                                                                                  |
+----------------------------------------------------------------------------------+
```

### 핵심 포인트

- 이 상태에서 사용자는 "무슨 질병이 문제인가"를 분명히 이해해야 한다.
- 상세로 들어가는 행동 유도가 자연스럽게 보여야 한다.

## D5. 상세 패널 열림 상태

### 목적

사용자가 현재 상태의 배경과 분포를 더 깊게 읽는 상태다.

### 구조 설명

- 오른쪽 상세 패널이 열린다.
- 기존 지도 위 요약 카드는 사라지거나 축약되고, 요약 정보가 패널 상단으로 흡수된다.
- 사용자는 지도와 상세 정보를 동시에 본다.

```text
+--------------------------------------------------------------------------------------------------+
| PRZM | 지역: 분당구 | 질병: A 질병 | 연령: 7세 | 기준 시점                                        |
+----------------------------------------------------------------------------------+---------------+
|                                                                                  | A 질병 상세   |
|                                                                                  |---------------|
|                                                                                  | 현재 위험 높음|
|                              MAIN MAP                                            | 최근 증가세   |
|                 [선택 지역 / 질병 확산 범위 / 위험도 색상 유지]                 |---------------|
|                                                                                  | 추세 차트     |
|                                                                                  | 속도 변화     |
|                                                                                  | 연령 분포     |
|                                                                                  | 성별 분포     |
|                                                                                  | 예측 보기     |
+----------------------------------------------------------------------------------+---------------+
```

### 핵심 포인트

- 상세 패널은 독립 페이지처럼 보이면 안 된다.
- 지도는 계속 살아 있어야 한다.
- 요약과 분석은 패널 안에서 자연스럽게 이어져야 한다.

## D6. 예측 확인 상태

### 목적

사용자가 현재 상태와 가까운 미래를 연결해 읽는 상태다.

### 구조 설명

- 오른쪽 패널의 하단 또는 별도 섹션에 예측이 이어진다.
- 예측은 `증가 / 정체 / 감소` 중심의 해석으로 먼저 보인다.

```text
+--------------------------------------------------------------------------------------------------+
| PRZM | 지역: 분당구 | 질병: A 질병 | 연령: 7세 | 기준 시점                                        |
+----------------------------------------------------------------------------------+---------------+
|                                                                                  | A 질병 상세   |
|                                                                                  |---------------|
|                              MAIN MAP                                            | 현재 위험 높음|
|                 [선택 상태 유지 / 지도는 계속 기준점 역할 수행]                 |---------------|
|                                                                                  | 추세 차트     |
|                                                                                  | 속도 변화     |
|                                                                                  | 연령 분포     |
|                                                                                  |---------------|
|                                                                                  | 1주 예측: 증가|
|                                                                                  | 1달 예측: 정체|
|                                                                                  | 신뢰도: 보통  |
+----------------------------------------------------------------------------------+---------------+
```

### 핵심 포인트

- 예측은 현재 상태의 연장선으로 보여야 한다.
- 숫자보다 해석 문장이 먼저 보여야 한다.

## 모바일 상태별 와이어프레임

## M1. 초기 진입 상태

### 목적

작은 화면에서도 사용자가 곧바로 현재 위험을 읽는 상태다.

### 구조 설명

- 상단에는 최소 제어 바가 있다.
- 화면 대부분은 지도다.
- 하단에는 작고 가벼운 요약 카드가 붙는다.

```text
+--------------------------------------+
| PRZM | 지역 | 질병 | 연령            |
+--------------------------------------+
|                                      |
|                                      |
|              MAIN MAP                |
|        (서울 / 경기 위험도)          |
|                                      |
|                                      |
|                                      |
|--------------------------------------|
| 현재 주의 질병                       |
| 서울 서남권 위험 높음                |
| 최근 확산세 커지는 중                |
+--------------------------------------+
```

### 핵심 포인트

- 요약 카드는 작게 시작해야 한다.
- 지도 가시성을 해치지 않아야 한다.

## M2. 지역/연령/질병 선택 상태

### 목적

모바일에서 선택 상태를 유지한 채 탐색을 이어가는 상태다.

### 구조 설명

- 상단 제어는 압축된 형태를 유지한다.
- 하단 카드 내용은 현재 선택 상태에 맞춰 갱신된다.

```text
+--------------------------------------+
| PRZM | 분당구 | A 질병 | 7세         |
+--------------------------------------+
|                                      |
|                                      |
|              MAIN MAP                |
|     [선택 지역 / 질병 범위 강조]     |
|                                      |
|                                      |
|                                      |
|--------------------------------------|
| 분당구 현재 위험 높음                |
| A 질병 확산 중                       |
| 상세 보기                            |
+--------------------------------------+
```

### 핵심 포인트

- 작은 화면에서도 현재 필터 상태가 보이지 않으면 안 된다.
- 하단 카드가 다음 행동의 진입점이 된다.

## M3. 상세 드로어 열림 상태

### 목적

사용자가 모바일에서 분석과 예측을 깊게 보는 상태다.

### 구조 설명

- 지도는 배경에 남는다.
- 하단 카드가 확장되어 드로어가 된다.
- 드로어 안에서 요약 → 분석 → 예측 순으로 정보가 이어진다.

```text
+--------------------------------------+
| PRZM | 분당구 | A 질병 | 7세         |
+--------------------------------------+
|                                      |
|              MAIN MAP                |
|      [배경에서 상태 유지되는 지도]    |
|                                      |
|--------------------------------------|
| ㅡ 드래그 핸들                        |
| A 질병 현재 위험 높음                |
| 최근 증가세                           |
|--------------------------------------|
| 추세 차트                             |
| 속도 변화                             |
| 연령 분포                             |
| 1주 예측: 증가                        |
| 1달 예측: 정체                        |
+--------------------------------------+
```

### 핵심 포인트

- 드로어는 패널의 역할을 대신하지만, 지도를 완전히 지우면 안 된다.
- 정보는 위에서 아래로 자연스럽게 읽혀야 한다.

## 상태 전환 규칙

- 데스크톱 기본 상태에서는 지도 위 요약 카드가 보인다.
- 데스크톱에서 상세 진입 시 오른쪽 패널이 열리고, 요약 정보는 패널 상단으로 이동한다.
- 모바일 기본 상태에서는 하단 카드가 보인다.
- 모바일에서 상세 진입 시 하단 카드가 확장 드로어로 전환된다.
- 상세 레이어를 닫아도 `지역 / 연령 / 질병` 선택 상태는 유지된다.
- 상태 전환은 화면 이동보다 같은 맥락 안의 확장처럼 느껴져야 한다.

## 와이어프레임 메모

- 이 문서는 고해상도 시각 디자인 문서가 아니다.
- 실제 UI 디자인 단계에서는 색상, 아이콘, 차트 형태, 지도 스타일, 모션이 추가 정의되어야 한다.
- 다만 정보 우선순위와 레이아웃 중심축은 이 문서를 벗어나지 않는 것이 좋다.
