# IA.md

## 문서 목적

이 문서는 PRZM의 정보 구조를 정의한다.

PRZM은 전통적인 다중 페이지 서비스보다, 지도 중심의 단일 화면 안에서 정보가 단계적으로 열리는 구조를 지향한다.

따라서 이 문서의 목적은 "페이지 목록"을 나열하는 것이 아니라, 사용자가 어떤 정보를 어떤 순서로 보고, 어디에서 더 깊은 정보를 열어보게 되는지를 정리하는 것이다.

## IA 원칙

- 지도는 제품의 중심이며, 다른 정보는 지도 탐색을 보조하거나 확장하는 역할을 한다.
- 사용자는 먼저 지역 위험을 보고, 그다음 연령과 질병 기준으로 좁혀 들어가며, 마지막으로 상세 분석과 예측을 확인한다.
- 최상위 구조는 단순해야 하며, 사용자가 길을 잃지 않아야 한다.
- 상세 정보는 별도 페이지로 이동시키기보다 현재 탐색 맥락을 유지한 채 열려야 한다.
- 데스크톱과 모바일은 같은 정보 구조를 공유하되, 표현 방식만 다르게 가져간다.

## 최상위 화면 구조

PRZM의 최상위 구조는 지도 단일 화면이다.

사용자는 앱에 진입하면 곧바로 메인 지도 화면에 도달한다.

이 메인 화면 안에서 정보는 크게 아래 영역으로 나뉜다.

1. 지도 영역
2. 상단 제어 영역
3. 요약 정보 영역
4. 상세 정보 레이어

별도의 `지도 / 분석 / 예측` 탭 구조를 두지 않는다.

분석과 예측은 지도에서 선택한 지역, 질병, 연령 상태를 기반으로 상세 정보 레이어 안에서 이어서 확인하는 구조로 둔다.

## 주요 정보 영역

### 1. 지도 영역

가장 큰 비중을 차지하는 핵심 영역이다.

사용자는 이 영역에서 아래 정보를 본다.

- 지역별 위험도
- 질병별 확산 범위
- 현재 선택된 지역 상태
- 현재 선택된 연령 기준 결과

지도 영역은 단순 배경이 아니라 탐색의 출발점이며, 대부분의 상호작용이 여기서 시작된다.

### 2. 상단 제어 영역

지도 위 또는 상단에 배치되는 제어 영역이다.

이 영역은 사용자가 현재 탐색 기준을 바꾸는 곳이다.

포함 정보와 기능:

- 지역 탐색 또는 현재 지역 기준 선택
- 질병 선택
- 연령 선택
- 기본 보기 전환에 필요한 최소 제어

이 영역은 사용자가 계속 볼 필요는 있지만, 지도 자체를 가리지 않도록 간결해야 한다.

### 3. 요약 정보 영역

현재 선택 상태를 짧고 명확하게 설명하는 영역이다.

주요 목적은 지도 해석을 돕는 것이다.

포함 정보:

- 현재 주의가 필요한 질병
- 선택 지역의 현재 위험 상태
- 최근 변화 방향
- 선택 연령 기준 핵심 요약

요약 정보는 데이터 테이블이 아니라, 사용자가 지금 무엇을 봐야 하는지 빠르게 이해시키는 해석 레이어에 가깝다.

### 4. 상세 정보 레이어

사용자가 특정 지역, 질병, 혹은 특정 상태를 더 깊게 보고 싶을 때 열리는 정보 레이어다.

이 레이어 안에서 아래 정보가 제공된다.

- 질병 상세 정보
- 기간별 추세
- 확산 속도 변화
- 연령별 분포
- 성별 분포
- 일주일 예측
- 한달 예측

상세 정보는 메인 화면의 맥락을 유지한 채 열려야 하므로, 별도 독립 페이지보다 패널/드로어 구조가 적합하다.

## 상태별 정보 구조

### 상태 1. 초기 진입

사용자는 메인 지도와 기본 요약 정보를 본다.

이 단계에서는 다음이 보여야 한다.

- 기본 지역 범위
- 현재 주요 위험 신호
- 기본 연령 기준 또는 추천 상태

초기 진입의 목적은 사용자가 "무엇을 해야 할지" 고민하지 않게 만드는 것이다.

### 상태 2. 지역 선택 후

사용자가 특정 지역을 선택하면, 지도와 요약 정보는 해당 지역 기준으로 갱신된다.

이 단계의 핵심은 "우리 동네"라는 질문에 더 가까워지는 것이다.

### 상태 3. 연령 선택 후

사용자가 0세부터 13세 사이의 연령을 선택하면, 현재 보여지는 위험도와 요약은 해당 연령 기준으로 다시 계산되어야 한다.

이 단계에서 정보 구조는 "전체 지역 정보"에서 "내 아이 연령 기준 정보"로 좁혀진다.

### 상태 4. 질병 선택 후

사용자가 특정 질병을 선택하면, 지도는 해당 질병의 확산 범위를 강조하고, 요약 정보 역시 그 질병 중심으로 바뀐다.

이 단계에서 사용자는 일반적인 위험이 아니라 구체적인 질병 단위로 상황을 파악한다.

### 상태 5. 상세 정보 열림

사용자가 더 깊은 정보를 원하면 상세 정보 레이어가 열린다.

이 단계에서는 선택된 지역, 연령, 질병 상태를 유지한 채 분석과 예측이 펼쳐져야 한다.

핵심은 사용자가 맥락을 잃지 않는 것이다.

## 반응형 구조

### 데스크톱

데스크톱에서는 상세 정보가 지도 오른쪽 패널로 열린다.

이 구조는 사용자가 지도를 계속 보면서 동시에 상세 내용을 읽을 수 있게 한다.

기본 구조:

- 중앙 또는 좌측 메인 지도
- 상단 제어 영역
- 지도 옆 요약/상세 패널

### 모바일

모바일에서는 상세 정보가 하단 드로어 형태로 열린다.

이 구조는 작은 화면에서 지도의 가시성을 최대한 보존하기 위한 것이다.

기본 구조:

- 전체 폭 지도
- 상단 또는 오버레이 제어 영역
- 하단 요약 카드
- 상향 확장 가능한 상세 드로어

데스크톱과 모바일은 정보의 종류는 같고, 열리는 방식만 다르다.

## 핵심 상호작용 흐름

PRZM의 기본 상호작용 흐름은 아래와 같다.

1. 사용자가 메인 지도에 진입한다.
2. 사용자가 지역을 확인하거나 선택한다.
3. 사용자가 아이의 연령을 선택한다.
4. 사용자가 현재 위험도와 확산 범위를 확인한다.
5. 사용자가 특정 질병을 선택한다.
6. 사용자가 상세 정보 레이어를 연다.
7. 사용자가 분석과 예측을 확인한다.

이 흐름은 깊어질수록 정보가 더 열리는 구조이지, 화면을 계속 갈아타는 구조가 아니다.

## 정보 계층 우선순위

PRZM에서 사용자가 우선적으로 보게 될 정보 계층은 아래 순서를 따른다.

1. 지금 어느 지역이 위험한가
2. 그 위험이 내 아이 연령대와 관련이 있는가
3. 어떤 질병이 퍼지고 있는가
4. 최근 변화 속도는 어떤가
5. 앞으로 위험이 커질까 줄어들까

이 순서를 거꾸로 만들지 않는다.

예를 들어 예측 차트나 상세 분포 정보가 첫 화면의 주요 정보보다 앞에 나오면 안 된다.

## 초기 범위 기준 IA

초기 정보 구조는 아래 범위를 기준으로 설계한다.

- 지역 범위: 서울특별시, 경기도
- 연령 범위: 0세부터 13세까지
- 주 사용자: 학부모, 일반 시민
- 핵심 질문: 우리 동네에서 지금 어떤 질병이 우리 아이 연령대에 퍼지고 있나

초기 IA는 전문가용 분석 도구의 복잡한 구조를 따르지 않는다.

핵심은 일반 시민이 낯선 데이터 구조를 배우지 않고도 즉시 상황을 이해할 수 있게 만드는 것이다.

## 제외 범위

현재 단계의 정보 구조에서 우선하지 않는 것은 아래와 같다.

- 복수의 상위 메뉴를 가진 복잡한 내비게이션
- 별도 보고서 페이지 중심 구조
- 고급 설정 메뉴 중심의 탐색 경험
- 행정 사용자 중심의 운영 대시보드 구조
- 사용자가 다수의 데이터 뷰를 동시에 비교하는 전문가형 레이아웃
