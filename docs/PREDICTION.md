# PREDICTION.md

## 문서 목적

이 문서는 PRZM의 예측 기능을 제품과 분석 관점에서 정의한다.

목적은 어떤 알고리즘을 쓸지 확정하는 것이 아니라, PRZM이 어떤 입력 데이터를 바탕으로 어떤 단위의 예측 결과를 만들고, 그 결과를 사용자에게 어떻게 해석해서 보여줄지를 정리하는 것이다.

초기 PRZM의 예측은 복잡한 수치 예보 서비스가 아니라, 일반 시민이 현재 위험의 가까운 미래를 이해하게 해주는 방향성 중심 기능이어야 한다.

## 예측 기능의 역할

PRZM의 예측 기능은 현재 상태를 넘어, 앞으로 위험이 더 커질지, 유지될지, 줄어들지를 이해하게 하는 역할을 한다.

사용자는 단순히 오늘의 상태만 알고 싶어하지 않는다. 지금 보이는 확산이 며칠 내 더 커질지, 일시적인지, 꺾이는 중인지 알고 싶어한다.

따라서 PRZM의 예측은 아래 질문에 답해야 한다.

- 앞으로 일주일 동안 이 질병 위험은 커질까 줄어들까
- 앞으로 한달 동안 이 확산은 이어질까 약해질까
- 현재 보이는 위험이 일시적인 신호인가, 더 지켜봐야 할 흐름인가

## 예측 설계 원칙

- 예측은 수치 자체보다 방향 해석이 우선이다.
- 예측은 현재 관측 데이터와 자연스럽게 이어져야 한다.
- 예측은 지역, 질병, 연령 맥락을 유지한 채 제공되어야 한다.
- 일반 시민이 이해하기 어려운 전문 모델 표현은 피한다.
- 예측은 확정 정보처럼 보이지 않아야 하며, 신뢰도와 한계를 함께 고려해야 한다.
- 초기 MVP에서는 일주일과 한달 예측에 집중한다.

## 입력 데이터

초기 PRZM의 예측은 아래 데이터에 기반한다.

### 1. 날짜별 관측값

특정 질병이 특정 지역에서 날짜별로 어떻게 관측되었는지에 대한 시계열 데이터다.

이 데이터는 예측의 가장 기본적인 입력값이다.

예시:

- 특정 질병의 최근 7일, 14일, 30일 변화
- 특정 지역에서의 증감 흐름

### 2. 지역 기준 관측값

예측은 지역 단위로 해석되어야 하므로, 특정 지역 안에서의 축적된 변화가 중요하다.

예시:

- 서울 내 특정 지역의 추세
- 경기도 내 특정 지역의 위험 변화

### 3. 질병 기준 관측값

질병마다 확산 패턴이 다르므로, 예측은 질병별로 분리되어야 한다.

예시:

- 같은 지역이라도 질병 A와 질병 B의 변화 방향이 다를 수 있음

### 4. 연령 기준 세부 집계

초기 PRZM은 어린이 0세부터 13세를 중심으로 보기 때문에, 예측도 연령 맥락을 반영해야 한다.

단, 초기 기본 단위는 `질병 × 지역 × 날짜`이며, 연령 정보는 해당 관측의 세부 집계로 연결된다.

예시:

- 특정 질병이 5세 집단에서 더 강하게 증가하는지 여부
- 전체 어린이 위험과 특정 연령 위험의 차이

### 5. 최근 변화량 및 추세 정보

단순 누적값보다 최근의 변화 속도와 방향이 예측에 더 중요하다.

예시:

- 최근 며칠간 급증 여부
- 증가 속도의 둔화 여부
- 안정화 또는 감소 전환 여부

## 예측 단위

초기 PRZM의 예측은 아래 단위를 기준으로 한다.

### 기본 단위

`질병 × 지역 × 기간`

즉, 예측은 "어떤 질병이 어떤 지역에서 앞으로 어떤 방향으로 갈 것인가"를 기본으로 본다.

### 조건부 단위

`질병 × 지역 × 연령 × 기간`

사용자가 특정 연령을 선택한 경우에는 해당 연령 기준 예측을 우선적으로 보여준다.

초기에는 예측의 가장 중요한 축이 질병과 지역이며, 연령은 그 결과를 더 개인화하는 조건으로 작동한다.

## 예측 기간

초기 PRZM은 두 개의 예측 기간을 제공한다.

## 1. 일주일 예측

### 의미

사용자가 가까운 미래의 즉각적인 위험 변화를 판단하는 데 쓰는 예측이다.

### 역할

- 당장 며칠 내 위험이 커지는지 판단
- 현재 위험이 더 확산되는 초기 신호인지 파악

## 2. 한달 예측

### 의미

사용자가 현재 흐름이 일시적 현상인지, 더 긴 추세로 이어지는지 판단하는 데 쓰는 예측이다.

### 역할

- 중기적인 확산 또는 소멸 방향 파악
- 지속적인 위험인지 일시적 변동인지 구분

초기 MVP에서는 더 긴 기간 예측은 다루지 않는다.

## 예측 결과 형식

초기 PRZM의 예측 결과는 방향 중심으로 표현한다.

### 핵심 방향 값

- `증가`
- `정체`
- `감소`

이 세 값은 사용자에게 가장 먼저 보여야 하는 핵심 결과다.

### 보조 값

방향 해석을 보완하기 위해 아래 값이 함께 붙을 수 있다.

- 예측 점수
- 신뢰도 수준
- 예측 기준 시점
- 짧은 해석 문장

### 예시 표현

- "앞으로 일주일 동안 위험 증가 가능성이 높음"
- "한달 기준으로는 현재 확산세가 완만해질 가능성이 있음"
- "현재 수준 유지 가능성이 높음"

핵심은 숫자 예보가 아니라 방향성 이해다.

## 사용자 노출 방식

예측은 상세 정보 레이어 안에서 현재 상태와 연결된 형태로 노출된다.

### 1. 예측은 상세 패널 내부에 위치한다

사용자는 먼저 현재 지역, 질병, 연령 기준 상황을 본 뒤 예측으로 들어간다.

예측이 메인 화면 첫 정보가 되어서는 안 된다.

### 2. 일주일 예측이 먼저 나온다

가까운 미래가 더 즉각적인 관심사이므로, 일주일 예측을 한달 예측보다 앞에 배치한다.

### 3. 해석 문장이 우선이다

사용자는 먼저 증가/정체/감소 방향과 짧은 요약 문장을 본다.

필요하면 보조 수치나 신뢰도를 추가로 볼 수 있다.

### 4. 현재 상태와 비교 가능해야 한다

예측은 현재 위험과 분리되어 보이지 않아야 한다.

예를 들어 사용자는 "현재 높음, 일주일 뒤 증가 가능성" 같은 식으로 현재와 미래를 연결해서 이해해야 한다.

## 신뢰도와 제한

예측은 관측과 다르게 본질적으로 불확실하다.

따라서 초기 PRZM은 예측을 아래 원칙으로 다뤄야 한다.

### 1. 확정 정보처럼 보이지 않게 한다

예측은 결과가 아니라 가능성의 방향이다.

### 2. 신뢰도를 보조 정보로 제공할 수 있다

예측 결과에 대해 높음, 보통, 낮음 같은 신뢰 수준을 붙일 수 있다.

### 3. 데이터 부족 구간은 예측을 약하게 표현한다

특정 지역, 질병, 연령 조합에서 데이터가 부족하면 예측 자체를 약하게 표시하거나 제한할 수 있다.

### 4. 예측은 설명 가능한 수준이어야 한다

사용자는 왜 이런 예측이 나왔는지 완전한 모델 설명까지는 원하지 않지만, 최근 추세를 바탕으로 한 방향성이라는 정도는 이해할 수 있어야 한다.

## 화면 및 기능 연결

예측 기능은 아래 화면과 기능에 연결된다.

### 메인 지도

- 직접적인 예측 정보는 최소화
- 필요 시 "위험 증가 신호" 정도의 힌트만 제공 가능

### 요약 정보 패널

- 현재 상태 중심
- 예측은 보조 문장으로 제한적으로 노출 가능

### 상세 정보 레이어

- 예측의 मुख्य 노출 위치
- 일주일 예측과 한달 예측 모두 여기서 제공

### 연령 필터

- 사용자가 특정 연령을 선택한 경우 예측도 해당 연령 기준으로 재해석

## 초기 범위와 제한

초기 PRZM의 예측 범위는 아래와 같다.

- 지역: 서울특별시, 경기도
- 질병: 초기 제품에서 지원하는 주요 질병
- 연령: 0세부터 13세까지 선택 가능
- 기간: 일주일, 한달
- 표현 방식: 방향 중심, 수치는 보조

현재 단계에서 이 문서는 아래 내용을 확정하지 않는다.

- 구체적인 모델 알고리즘
- 머신러닝 사용 여부
- 외부 변수 반영 방식
- 성능 지표 계산 방식

## 확장 방향

향후 예측 기능은 아래 방향으로 발전할 수 있다.

### 1. 예측 설명 강화

- 최근 변화 근거를 더 명확히 제시
- 예측 결과의 이유를 더 잘 설명

### 2. 예측 범위 확장

- 전국 단위 확대
- 전 연령층 확대

### 3. 예측 표현 고도화

- 방향 외 보조 수치 강화
- 지역 비교 예측
- 질병 간 상대 위험 비교

### 4. 신뢰도 표현 정교화

- 데이터 충분성에 따른 예측 강도 차등
- 조건에 따른 예측 제한 처리
