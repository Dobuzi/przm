# COMPONENTS.md

## 문서 목적

이 문서는 PRZM의 MVP UI를 구성하는 주요 컴포넌트를 정의한다.

목적은 모든 디자인 시스템 요소를 완성하는 것이 아니라, PRZM에서 반복적으로 사용되는 핵심 UI 단위를 제품 구조와 함께 정리하는 것이다.

이 문서는 `WIREFRAMES.md`, `SCREENS.md`, `UI.md`를 바탕으로 작성하며, 이후 실제 프론트엔드 컴포넌트 설계와 디자인 시스템 정리에 기준점으로 사용한다.

## 컴포넌트 설계 원칙

- 컴포넌트는 PRZM의 핵심 질문에 답하는 경험을 더 빠르고 명확하게 만들어야 한다.
- 범용성은 중요하지만, 지금 단계에서는 MVP에 필요한 범위만 정의한다.
- 지도 중심 제품이므로, 많은 컴포넌트는 지도를 보조하거나 지도와 함께 작동해야 한다.
- 상태 정보는 사용자가 즉시 이해할 수 있어야 하며, 과도한 장식보다 명확한 상태 전달이 우선이다.
- 데스크톱과 모바일은 같은 컴포넌트 역할을 공유하되, 표현 형태는 달라질 수 있다.

## Foundation

## 1. Color Scale Token

### 역할

위험도와 상태 강도를 일관되게 표현하는 기본 색상 체계다.

### 구성

- 낮은 위험 단계
- 중간 위험 단계
- 높은 위험 단계
- 강조 상태
- 비활성 상태

### 상태

- normal
- selected
- muted

### 사용 화면

- 메인 지도
- 요약 카드
- 상세 패널

### 주의점

- 위험도 색상은 장식이 아니라 핵심 정보 표현 수단이다.
- 단계 수는 너무 많지 않아야 한다.

## 2. Typography Token

### 역할

정보 계층을 일관되게 유지하는 텍스트 체계다.

### 구성

- 페이지/패널 제목
- 섹션 제목
- 핵심 상태 문장
- 보조 설명
- 메타 정보

### 사용 화면

- 전체 화면 공통

### 주의점

- 핵심 문장은 짧고 빠르게 읽혀야 한다.
- 숫자보다 해석 문장이 먼저 보이도록 계층을 구성한다.

## 3. Spacing & Radius Token

### 역할

카드, 패널, 제어 요소의 간격과 형태를 일관되게 만드는 기본 단위다.

### 구성

- 작은 간격
- 기본 간격
- 큰 간격
- 카드 라운드
- 패널 라운드

### 사용 화면

- 전체 화면 공통

### 주의점

- 지도 제품이므로 간격이 과도하게 넓어지면 정보 밀도가 떨어질 수 있다.

## Navigation & Controls

## 4. Top Control Bar

### 역할

현재 탐색 기준을 보여주고, 지역/질병/연령 선택을 제어하는 상단 공통 컴포넌트다.

### 구성

- 브랜드 또는 서비스명
- 지역 선택 진입점
- 질병 선택 진입점
- 연령 선택 진입점
- 기준 시점 표시

### 상태

- default
- sticky
- compact

### 사용 화면

- 데스크톱 전체 상태
- 모바일 전체 상태

### 주의점

- 항상 접근 가능해야 하지만, 지도를 과하게 가리면 안 된다.
- 선택 상태는 축약되더라도 분명히 보여야 한다.

## 5. Filter Chip

### 역할

현재 선택된 지역, 질병, 연령 상태를 짧게 표시하거나 빠르게 바꾸는 컴포넌트다.

### 구성

- 라벨
- 값
- 선택/닫기 액션

### 상태

- default
- selected
- removable

### 사용 화면

- 상단 제어 바
- 모바일 요약 카드
- 상세 패널 상단

### 주의점

- 필터 상태는 보조 정보가 아니라 전체 해석 기준이다.

## 6. Selector Sheet / Dropdown

### 역할

지역, 질병, 연령을 선택하는 실제 선택 UI다.

### 구성

- 목록 또는 검색
- 최근 선택 상태
- 적용 액션

### 상태

- closed
- open
- active selection

### 사용 화면

- 상단 제어 바
- 모바일 선택 시트

### 주의점

- 데스크톱은 드롭다운형, 모바일은 시트형이 자연스럽다.
- 연령 선택은 빠르게 조작 가능해야 한다.

## Map Layer Components

## 7. Risk Map Layer

### 역할

지역별 위험도를 색상 단계로 표현하는 지도 레이어다.

### 구성

- 지역 경계
- 위험도 색상 채움
- 선택 지역 강조

### 상태

- default
- hovered
- selected
- filtered

### 사용 화면

- 메인 지도
- 지역 선택 상태
- 연령 적용 상태

### 주의점

- 위험도는 한눈에 읽혀야 한다.
- 선택 상태와 위험도 표현이 충돌하면 안 된다.

## 8. Spread Overlay

### 역할

특정 질병 선택 시 확산 범위나 집중 영역을 강조하는 지도 오버레이다.

### 구성

- 범위 강조 영역
- 밀집도 또는 강조 패턴
- 중심 지점 표시

### 상태

- hidden
- visible
- active disease

### 사용 화면

- 질병 선택 상태
- 상세 패널 열림 상태

### 주의점

- 기본 위험도 레이어를 완전히 덮으면 안 된다.
- 사용자는 현재 위험도와 질병 확산 범위를 함께 읽을 수 있어야 한다.

## 9. Map Legend

### 역할

위험도 색상 또는 지도 표현의 의미를 짧게 설명하는 보조 컴포넌트다.

### 구성

- 색상 단계
- 간단한 라벨

### 상태

- collapsed
- expanded

### 사용 화면

- 메인 지도
- 질병 선택 상태

### 주의점

- 범례는 필요하지만, 사용자 주의를 빼앗는 중심 요소가 되면 안 된다.

## Summary & Analysis Components

## 10. Summary Card

### 역할

현재 가장 중요한 위험 신호를 짧은 문장으로 요약하는 핵심 컴포넌트다.

### 구성

- 상태 헤드라인
- 주요 질병
- 현재 지역 해석
- 변화 방향
- 상세 보기 진입점

### 상태

- default
- regional
- age-filtered
- disease-focused

### 사용 화면

- 데스크톱 초기 지도 상태
- 데스크톱 지역/연령/질병 상태
- 모바일 하단 카드

### 주의점

- 문장은 짧고 강해야 한다.
- 긴 설명이나 많은 수치를 넣으면 안 된다.

## 11. Insight Row

### 역할

요약 패널이나 상세 패널 안에서 핵심 인사이트를 한 줄 단위로 보여주는 컴포넌트다.

### 구성

- 라벨
- 값 또는 해석
- 보조 설명

### 상태

- default
- caution
- positive
- neutral

### 사용 화면

- 요약 카드
- 상세 패널 상단

### 주의점

- 여러 줄이 쌓여도 스캔이 쉬워야 한다.

## 12. Trend Chart Block

### 역할

기간별 추세와 변화 흐름을 보여주는 분석 컴포넌트다.

### 구성

- 섹션 제목
- 간단한 추세 차트
- 보조 해석 문장

### 상태

- loading
- ready
- empty

### 사용 화면

- 상세 패널
- 모바일 상세 드로어

### 주의점

- 차트만 있고 해석이 없으면 안 된다.
- 차트는 분석 보조 수단이지 첫 화면 핵심 요소가 아니다.

## 13. Distribution Block

### 역할

연령별 또는 성별 분포를 보여주는 분석 블록이다.

### 구성

- 섹션 제목
- 분포 시각화
- 주요 해석 문장

### 상태

- age mode
- gender mode
- empty

### 사용 화면

- 상세 패널
- 모바일 상세 드로어

### 주의점

- 초기 MVP에서는 너무 많은 세부 축을 동시에 보여주지 않는다.

## 14. Forecast Card

### 역할

일주일 또는 한달 예측 결과를 방향 중심으로 보여주는 컴포넌트다.

### 구성

- 예측 기간
- 방향 값
- 짧은 해석 문장
- 신뢰도

### 상태

- increasing
- steady
- decreasing
- low confidence

### 사용 화면

- 상세 패널 예측 섹션
- 모바일 상세 드로어 예측 섹션

### 주의점

- 숫자보다 방향과 해석이 먼저 보여야 한다.
- 현재 상태와 분리된 카드처럼 느껴지면 안 된다.

### 구성

- 패널 헤더
- 요약 영역
- 분석 섹션
- 예측 섹션
- 닫기 액션

### 상태

- closed
- open
- scrolled

### 사용 화면

- 데스크톱 상세 상태
- 데스크톱 예측 상태

### 구성

- 예측 섹션

### 상태

- half-open
- expanded

- 제목

- 지역 선택
- 질병 선택
- 연령 선택

### 주의점

- 핵심 흐름을 막는 무거운 모달 남용은 피한다.

### 구성

- 관측 기준 시점
- 예측 생성 시점

### 상태

- compact
- detailed

### 사용 화면

- 상단 제어 바
- 상세 패널

### 주의점

- 신뢰를 위한 정보이지만, 메인 메시지를 압도하면 안 된다.

## 컴포넌트 간 관계

PRZM의 컴포넌트는 아래 흐름으로 연결된다.

- `Top Control Bar`와 `Filter Chip`이 현재 탐색 상태를 정의한다.
- `Risk Map Layer`와 `Spread Overlay`가 현재 상태를 지도에서 시각화한다.
- `Summary Card`가 현재 상태를 해석한다.
- `Detail Side Panel` 또는 `Bottom Detail Drawer`가 요약을 흡수해 분석과 예측으로 확장한다.
- `Trend Chart Block`, `Distribution Block`, `Forecast Card`가 상세 이해를 구성한다.
- `Loading State Block`, `Empty State Block`, `Timestamp Meta`가 신뢰성과 사용성을 보완한다.

## MVP 범위

초기 MVP에서 우선 정의하고 구현해야 하는 컴포넌트는 아래와 같다.

- `Top Control Bar`
- `Filter Chip`
- `Risk Map Layer`
- `Spread Overlay`
- `Summary Card`
- `Trend Chart Block`
- `Distribution Block`
- `Forecast Card`
- `Detail Side Panel`
- `Bottom Detail Drawer`
- `Loading State Block`
- `Empty State Block`
- `Timestamp Meta`

## 제외 범위

현재 단계에서 우선하지 않는 컴포넌트는 아래와 같다.

- 복수 지역 비교 전용 컴포넌트
- 고급 리포트 생성 컴포넌트
- 개인화 프로필 컴포넌트
- 알림 센터 전용 컴포넌트
- 전문가용 데이터 테이블 컴포넌트
