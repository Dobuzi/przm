# PRODUCT.md

## 제품 한 줄 정의

PRZM은 한국어 "퍼짐"의 발음을 바탕으로 만든 이름으로, 우리 동네에서 어떤 질병이 우리 아이 연령대에 퍼지고 있는지를 지도 중심으로 빠르게 이해하게 해주는 질병 확산 모니터링 웹 앱이다.

`GOAL.md`가 제품의 방향과 범위를 정의한다면, 이 문서는 그 목표를 어떤 사용자 경험과 제품 구조로 풀어낼지를 정리한다.

## 목표 사용자

PRZM의 1차 사용자는 학부모와 일반 시민이다.

이들은 방역 전문가나 행정 담당자가 아니며, 복잡한 통계 리포트를 읽기보다 내 주변의 위험 신호를 빠르게 이해하고 싶어한다.

특히 초기 제품에서는 서울특별시와 경기도에 거주하거나 해당 지역의 상황을 확인하려는 보호자를 핵심 사용자로 본다.

## 사용자의 핵심 질문

PRZM이 가장 먼저 답해야 하는 질문은 아래와 같다.

> 우리 동네에서 지금 어떤 질병이 우리 아이 연령대에 퍼지고 있나?

이 질문에 빠르게 답할 수 있어야 사용자는 PRZM을 다시 찾게 된다.

## 해결하려는 문제

현재 질병 관련 정보는 보통 뉴스, 공공 통계, 병원 정보, 커뮤니티 경험담 등으로 흩어져 있다.

이 정보들은 다음과 같은 한계를 가진다.

- 지역 단위로 체감 가능한 위험을 바로 알기 어렵다.
- 특정 연령대, 특히 어린이 연령별 차이를 세밀하게 보기 어렵다.
- 현재 상황은 보여줘도 확산 방향과 속도는 이해하기 어렵다.
- 일반 시민이 지도 위에서 직관적으로 판단하기 어려운 형태로 제공되는 경우가 많다.

PRZM은 이 문제를 해결하기 위해 질병 확산 데이터를 지역, 시간, 연령 기준으로 재구성하고, 이를 지도 중심 경험으로 제공한다.

## 핵심 사용자 시나리오

### 시나리오 1. 우리 동네 위험 확인

사용자는 앱에 들어오자마자 지도를 본다.

지도에서 자신의 지역 또는 관심 지역을 찾고, 현재 위험도가 높아진 질병과 그 확산 범위를 확인한다.

이 단계에서 사용자는 "지금 어디가 위험한가"를 빠르게 파악할 수 있어야 한다.

### 시나리오 2. 우리 아이 연령대 확인

사용자는 아이의 연령을 선택하거나 기본 설정된 연령 필터를 적용한다.

그 결과 지도와 요약 정보는 해당 연령대 기준으로 다시 표현된다.

이 단계에서 사용자는 "전체 유행"이 아니라 "내 아이와 가까운 위험"을 확인할 수 있어야 한다.

### 시나리오 3. 질병별 상세 분석 확인

사용자는 특정 질병 또는 특정 지역을 선택해 상세 분석 탭으로 이동한다.

상세 분석 탭에서는 확산 속도, 최근 변화 추세, 연령별 분포, 성별 분포, 단기 예측을 확인한다.

이 단계에서 사용자는 단순 조회를 넘어서 "위험이 커지고 있는지, 줄어들고 있는지"를 이해할 수 있어야 한다.

### 시나리오 4. 가까운 미래 예측 확인

사용자는 현재 데이터뿐 아니라 앞으로 일주일, 한달 동안의 확산과 소멸 가능성을 본다.

예측은 복잡한 모델 설명보다 사용자에게 의미 있는 방향성으로 전달되어야 한다. 예를 들어 위험 증가, 정체, 감소 같은 해석이 우선이다.

## 핵심 화면 구조

### 1. 지도 중심 홈 화면

첫 화면은 지도다.

이 화면의 가장 중요한 역할은 사용자가 자신의 지역에서 현재 위험도와 확산 범위를 빠르게 이해하게 하는 것이다.

홈 화면에는 아래 정보가 우선적으로 표현되어야 한다.

- 지역별 위험도
- 질병별 확산 영역
- 최근 확산 변화 신호
- 선택된 연령대 기준 정보

지도는 제품의 배경이 아니라 핵심 인터페이스다.

### 2. 요약 정보 패널

지도와 함께 현재 선택된 지역과 연령 기준의 핵심 요약을 보여준다.

예를 들어 다음과 같은 정보가 들어갈 수 있다.

- 지금 주의가 필요한 질병
- 최근 일주일 변화
- 위험 상승 지역 여부
- 선택 연령대에서 두드러지는 질병

이 패널은 사용자가 지도만으로 놓칠 수 있는 핵심 문장을 보완한다.

### 3. 상세 분석 탭

상세 분석 탭은 사용자가 특정 질병이나 지역을 깊게 볼 때 사용하는 화면이다.

이 탭은 아래 내용을 포함한다.

- 기간별 확산 추세
- 확산 속도 변화
- 연령별 분포
- 성별 분포
- 향후 일주일 및 한달 예측

지도에서 상황을 파악하고, 상세 분석 탭에서 이유와 변화 방향을 읽는 구조를 기본 흐름으로 둔다.

## 핵심 기능 범위

초기 제품에서 PRZM이 반드시 제공해야 하는 기능은 아래와 같다.

### 모니터링

- 서울/경기도 지역 질병 확산 현황 표시
- 어린이 0세부터 13세까지 1세 단위 필터링
- 지역 선택 및 탐색

### 분석

- 확산 영역 확인
- 확산 속도 확인
- 연령별 차이 확인
- 성별 차이 확인

### 예측

- 향후 일주일 예측
- 향후 한달 예측
- 확산 또는 소멸 방향성 제시

## 초기 범위

초기 버전은 서울특별시와 경기도를 중심으로 동작한다.

초기 타깃 연령은 0세부터 13세까지의 어린이이며, 1세 단위 분석을 기본으로 한다.

초기 제품은 일반 시민이 빠르게 이해할 수 있는 수준의 지도 기반 정보 경험을 우선한다.

## 초기 제외 범위

초기 버전에서 당장 우선하지 않는 범위는 아래와 같다.

- 전국 단위 전체 지역 커버리지
- 전 연령층 동시 정밀 분석
- 전문가용 복잡 리포트 작성 기능
- 행정용 운영 대시보드 성격의 기능
- 사용자가 직접 데이터를 입력하거나 수정하는 기능

## 제품 원칙

- 사용자는 먼저 지도로 이해하고, 필요할 때 상세 분석으로 들어간다.
- 정보는 현재 상태뿐 아니라 변화 방향까지 보여줘야 한다.
- 분석 단위는 일반 시민에게 의미 있는 수준으로 단순해야 한다.
- 초기에는 어린이 중심 문제를 깊게 푸는 것이 넓은 범위를 얕게 다루는 것보다 중요하다.
- 예측은 숫자 나열보다 행동 가능한 해석을 우선해야 한다.

## 성공 기준

초기 제품이 성공적이라고 볼 수 있는 기준은 아래와 같다.

- 사용자가 첫 화면에서 자신의 지역 위험도를 빠르게 이해할 수 있다.
- 사용자가 특정 연령대 기준으로 위험을 쉽게 다시 볼 수 있다.
- 사용자가 어떤 질병이 퍼지는 중인지와 그 범위를 직관적으로 파악할 수 있다.
- 사용자가 앞으로 위험이 커질지 줄어들지를 이해할 수 있다.

이 문서는 이후 정보 구조, UI 설계, 데이터 정의, 예측 로직 문서의 기준점으로 사용한다.
