# SCREENS.md

## 문서 목적

이 문서는 PRZM의 MVP 화면을 상태 기반으로 정의한다.

PRZM은 여러 개의 독립 페이지를 오가는 구조가 아니라, 지도 중심의 단일 화면 안에서 정보가 단계적으로 열리는 제품이다.

따라서 이 문서에서 말하는 "화면"은 URL 단위 페이지라기보다, 사용자의 선택 상태에 따라 달라지는 주요 화면 상태를 의미한다.

## 화면 정의 원칙

- 기준 서술은 데스크톱 화면을 중심으로 한다.
- 모바일은 같은 정보 구조를 유지하되, 표현 방식과 레이어 구조를 별도로 정의한다.
- MVP에서는 화면 수를 늘리기보다, 한 화면 안의 상태 변화를 명확히 정의하는 것이 중요하다.
- 각 화면 상태는 사용자의 목표와 직접 연결되어야 한다.
- 상세 정보는 별도 전환 페이지가 아니라 패널 또는 드로어로 열린다.

## 공통 화면 구조

PRZM의 모든 핵심 화면 상태는 아래 공통 구조를 바탕으로 한다.

### 공통 영역

- 상단 제어 영역
- 메인 지도 영역
- 요약 정보 영역
- 상세 정보 레이어

### 공통 상태 정보

모든 화면 상태에서 사용자는 아래 선택 상태를 확인할 수 있어야 한다.

- 현재 보고 있는 지역
- 현재 선택된 연령
- 현재 선택된 질병
- 현재 데이터 기준 시점

### 공통 인터랙션

- 지도 이동 및 확대/축소
- 지역 선택
- 연령 선택
- 질병 선택
- 상세 정보 열기 및 닫기

## 데스크톱 기준 핵심 화면 상태

## 화면 1. 초기 진입 상태

### 목적

사용자가 앱에 들어오자마자 현재 주요 위험 상황을 빠르게 이해하도록 한다.

### 진입 조건

- 첫 방문
- 재방문 시 기본 상태 진입

### 주요 구성

- 상단 제어 바
- 서울/경기도 범위의 메인 지도
- 기본 위험도 시각화
- 요약 정보 패널

### 보여야 하는 정보

- 현재 주의가 필요한 질병
- 지역별 위험도 개요
- 기본 연령 기준 또는 추천 상태
- 최근 변화 방향

### 사용자 행동

- 지도를 훑어본다
- 지역을 선택한다
- 연령을 바꾼다
- 질병을 선택한다

## 화면 2. 지역 선택 상태

### 목적

사용자가 "우리 동네" 수준으로 정보를 좁혀 현재 위험을 확인하게 한다.

### 진입 조건

- 사용자가 지도에서 특정 지역을 선택함

### 주요 구성

- 선택 지역이 강조된 메인 지도
- 선택 지역 기준 요약 정보
- 동일한 상단 제어 바

### 보여야 하는 정보

- 선택 지역의 현재 위험도
- 선택 지역에서 주의할 질병
- 선택 지역 기준 최근 변화

### 사용자 행동

- 해당 지역의 위험 상태를 확인한다
- 연령을 선택해 더 좁힌다
- 특정 질병을 선택해 상세로 들어간다

## 화면 3. 연령 필터 적용 상태

### 목적

사용자가 전체 어린이 집단이 아니라 특정 연령 기준으로 위험을 다시 해석하게 한다.

### 진입 조건

- 사용자가 0세부터 13세 사이 연령을 선택함

### 주요 구성

- 연령 기준이 반영된 지도
- 연령 기준 요약 정보
- 현재 선택 연령이 명확히 드러나는 제어 바

### 보여야 하는 정보

- 해당 연령에서의 위험도 변화
- 연령 기준 주요 질병
- 전체 상태 대비 달라진 위험 신호

### 사용자 행동

- 다른 연령과 비교하듯 재선택한다
- 특정 질병을 선택한다
- 상세 정보를 연다

## 화면 4. 질병 선택 상태

### 목적

사용자가 특정 질병을 중심으로 확산 범위와 위험도를 이해하게 한다.

### 진입 조건

- 사용자가 질병을 선택함

### 주요 구성

- 선택 질병의 확산 범위가 강조된 지도
- 질병 중심 요약 정보
- 선택된 지역/연령/질병 상태가 반영된 상단 제어 바

### 보여야 하는 정보

- 해당 질병의 확산 영역
- 선택 지역에서의 해당 질병 위험
- 최근 변화 방향

### 사용자 행동

- 질병 중심으로 상황을 본다
- 상세 패널을 연다
- 예측까지 확인한다

## 화면 5. 상세 정보 패널 열림 상태

### 목적

사용자가 특정 지역, 연령, 질병에 대한 상세 분석을 현재 맵 맥락 안에서 확인하게 한다.

### 진입 조건

- 사용자가 상세 보기 액션을 수행함

### 주요 구성

- 좌측 또는 중앙의 메인 지도
- 오른쪽 고정 상세 패널
- 패널 안의 분석 섹션

### 보여야 하는 정보

- 기간별 추세
- 확산 속도 변화
- 연령별 분포
- 성별 분포

### 사용자 행동

- 현재 위험의 배경을 이해한다
- 예측 섹션까지 스크롤하거나 이동한다
- 패널을 닫고 다시 지도 탐색으로 돌아간다

## 화면 6. 예측 확인 상태

### 목적

사용자가 현재 상태에서 가까운 미래의 확산 또는 소멸 방향을 이해하게 한다.

### 진입 조건

- 상세 패널 안에서 예측 섹션에 진입함

### 주요 구성

- 계속 유지되는 메인 지도
- 오른쪽 상세 패널 내 예측 섹션

### 보여야 하는 정보

- 앞으로 일주일 예측
- 앞으로 한달 예측
- 증가, 정체, 감소 해석

### 사용자 행동

- 단기와 중기 흐름을 확인한다
- 현재 상태와 예측 상태를 함께 해석한다
- 다시 지도 또는 다른 질병/연령 선택으로 이동한다

## 모바일 기준 핵심 화면 상태

모바일은 정보 구조는 같지만, 공간 제약 때문에 레이어 방식이 다르다.

## 화면 M1. 초기 진입 상태

### 주요 특징

- 전체 폭 지도
- 상단 또는 오버레이 제어 바
- 하단 요약 카드

### 차이점

- 데스크톱처럼 넓은 요약 패널 대신, 핵심 문장 중심의 카드가 먼저 보인다.

## 화면 M2. 지역/연령/질병 선택 상태

### 주요 특징

- 선택 상태가 지도 위와 하단 카드에 동시에 반영된다.
- 사용자는 작은 화면에서도 현재 필터 상태를 명확히 볼 수 있어야 한다.

### 차이점

- 선택 UI는 드롭다운보다 시트, 칩, 피커 형태가 더 적합할 수 있다.

## 화면 M3. 상세 드로어 열림 상태

### 주요 특징

- 하단에서 올라오는 드로어가 상세 정보를 담는다.
- 지도는 배경에서 계속 유지된다.

### 포함 정보

- 추세
- 속도 변화
- 분포
- 예측

### 차이점

- 데스크톱의 오른쪽 패널 기능을 하단 드로어가 대신한다.
- 드로어는 부분 확장, 전체 확장, 닫기 상태를 가질 수 있다.

## 상태 전환 규칙

- 초기 진입에서 지역 선택으로 이동할 수 있다.
- 초기 진입 또는 지역 선택 상태에서 연령 선택으로 이동할 수 있다.
- 지역/연령 상태에서 질병 선택 상태로 이동할 수 있다.
- 질병 선택 상태에서 상세 정보 상태로 이동할 수 있다.
- 상세 정보 상태 안에서 예측 확인 상태로 자연스럽게 이어진다.
- 사용자는 언제든 상세 레이어를 닫고 지도 중심 상태로 돌아올 수 있어야 한다.

상태 전환은 가능한 한 비파괴적이어야 한다.

즉, 사용자가 상세 패널을 닫더라도 선택한 지역, 연령, 질병 상태는 유지되는 것이 기본이다.

## MVP 화면 범위

MVP에서 반드시 정의하고 구현해야 하는 화면 상태는 아래와 같다.

- 초기 진입 상태
- 지역 선택 상태
- 연령 필터 적용 상태
- 질병 선택 상태
- 상세 정보 열림 상태
- 예측 확인 상태

모바일에서는 이 상태들을 각각 독립 페이지로 만들기보다, 지도 + 카드/드로어 조합으로 표현한다.

## 제외 범위

현재 단계에서 우선하지 않는 화면은 아래와 같다.

- 별도의 로그인/회원가입 화면
- 개인화 설정 전용 페이지
- 전문가용 대시보드 화면
- 복수 지역 비교 전용 화면
- 고급 리포트 전용 화면
