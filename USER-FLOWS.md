# USER-FLOWS.md

## 문서 목적

이 문서는 PRZM의 핵심 사용자 흐름을 정의한다.

`IA.md`가 정보 구조를 설명한다면, 이 문서는 사용자가 어떤 목표를 가지고 앱에 들어와 어떤 단계를 거쳐 원하는 정보를 얻는지를 정리한다.

초기 PRZM은 복잡한 다기능 서비스가 아니라, 지도 중심의 빠른 이해와 점진적 탐색을 제공하는 제품이다. 따라서 사용자 흐름 역시 짧고 명확해야 한다.

## 핵심 사용자와 핵심 질문

PRZM의 1차 사용자는 학부모와 일반 시민이다.

이 사용자가 PRZM에 들어오는 가장 중요한 이유는 아래 질문에 답하기 위해서다.

> 우리 동네에서 지금 어떤 질병이 우리 아이 연령대에 퍼지고 있나?

모든 핵심 사용자 흐름은 이 질문을 중심으로 설계한다.

## 메인 사용자 플로우

## 플로우 1. 우리 동네 현재 위험 확인

### 목표

사용자는 자신의 지역에서 현재 어떤 질병 위험이 존재하는지 빠르게 알고 싶다.

### 시작점

사용자가 PRZM 메인 지도 화면에 진입한다.

### 트리거

- 앱 첫 진입
- 다시 방문해 현재 상황을 재확인하려는 경우

### 단계

1. 사용자는 메인 지도 화면을 본다.
2. 사용자는 현재 기본 지역 또는 관심 지역을 확인한다.
3. 지도에서 지역별 위험도와 확산 범위를 훑어본다.
4. 요약 정보 영역에서 현재 주의가 필요한 질병과 변화 방향을 읽는다.
5. 필요하면 특정 지역을 선택해 해당 지역 기준으로 내용을 좁혀 본다.

### 성공 조건

- 사용자가 우리 동네가 현재 어느 정도 위험한지 이해한다.
- 사용자가 어떤 질병이 눈에 띄는지 파악한다.
- 사용자가 더 깊이 볼 필요가 있는지 아닌지 판단할 수 있다.

### 핵심 정보

- 지역별 위험도
- 현재 주의 질병
- 확산 범위
- 최근 변화 방향

### UX 원칙

- 첫 진입 후 몇 초 안에 핵심 상황을 파악할 수 있어야 한다.
- 사용자는 복잡한 조작 없이도 기본 정보를 얻을 수 있어야 한다.

## 보조 사용자 플로우

## 플로우 2. 내 아이 연령대로 좁혀 보기

### 목표

사용자는 전체 어린이 데이터가 아니라, 내 아이 나이와 더 가까운 위험을 보고 싶다.

### 시작점

사용자가 메인 지도 화면 또는 특정 지역 선택 상태에 있다.

### 트리거

- 아이의 정확한 나이에 맞는 위험을 확인하고 싶은 경우
- 전체 어린이 위험과 특정 연령 위험이 다를 수 있다고 느끼는 경우

### 단계

1. 사용자는 연령 선택 UI를 연다.
2. 0세부터 13세 사이에서 아이의 연령을 선택한다.
3. 지도와 요약 정보가 해당 연령 기준으로 갱신된다.
4. 사용자는 위험도, 확산 범위, 주의 질병이 어떻게 달라졌는지 확인한다.

### 성공 조건

- 사용자가 선택한 연령 기준으로 결과가 바뀌었다는 점을 분명히 인지한다.
- 사용자가 전체 위험과 자신의 상황 사이 차이를 이해한다.

### 핵심 정보

- 연령 기준 위험도 변화
- 연령 기준 질병 분포 차이
- 선택 연령 상태 표시

### UX 원칙

- 연령 변경은 즉시 반영되어야 한다.
- 사용자는 현재 어떤 연령 기준으로 보고 있는지 항상 알 수 있어야 한다.

## 플로우 3. 특정 질병 상세 보기

### 목표

사용자는 현재 눈에 띄는 특정 질병을 더 깊게 이해하고 싶다.

### 시작점

사용자가 메인 지도에서 위험 신호 또는 질병 요약을 확인한 상태다.

### 트리거

- 특정 질병이 지도나 요약 패널에서 두드러져 보이는 경우
- 사용자가 뉴스나 주변 경험으로 특정 질병을 이미 의식하고 있는 경우

### 단계

1. 사용자는 특정 질병을 선택한다.
2. 지도는 해당 질병의 확산 범위를 강조해서 보여준다.
3. 요약 정보는 해당 질병 중심으로 갱신된다.
4. 사용자는 상세 정보 레이어를 연다.
5. 상세 레이어에서 추세, 확산 속도, 연령 분포, 성별 분포를 확인한다.

### 성공 조건

- 사용자가 특정 질병이 왜 중요한지 이해한다.
- 사용자가 이 질병이 넓어지는지 줄어드는지 판단할 수 있다.

### 핵심 정보

- 질병별 확산 영역
- 최근 추세
- 확산 속도 변화
- 연령 및 성별 분포

### UX 원칙

- 질병 선택 전후의 차이가 명확해야 한다.
- 상세 정보는 지도 맥락을 잃지 않은 상태에서 열려야 한다.

## 플로우 4. 가까운 미래 예측 보기

### 목표

사용자는 현재 상황뿐 아니라 앞으로 위험이 더 커질지 줄어들지를 알고 싶다.

### 시작점

사용자가 특정 지역, 연령, 질병을 선택한 상태다.

### 트리거

- 지금 상황이 일시적 현상인지 확인하고 싶은 경우
- 며칠 혹은 몇 주 후 위험도를 미리 가늠하고 싶은 경우

### 단계

1. 사용자는 상세 정보 레이어를 연다.
2. 예측 섹션으로 이동한다.
3. 앞으로 일주일 예측을 확인한다.
4. 앞으로 한달 예측을 확인한다.
5. 위험 증가, 정체, 감소 중 어떤 방향인지 해석한다.

### 성공 조건

- 사용자가 단기와 중기 흐름을 구분해 이해한다.
- 사용자가 앞으로 더 주의해야 하는지, 안정되는지 감을 잡는다.

### 핵심 정보

- 일주일 예측
- 한달 예측
- 확산/정체/소멸 방향

### UX 원칙

- 예측은 복잡한 모델 설명보다 해석 가능한 방향 정보가 우선이다.
- 현재 상태와 예측 상태가 자연스럽게 이어져 보이도록 구성해야 한다.

## 빈 상태 및 예외 흐름

## 흐름 A. 데이터가 충분하지 않은 경우

### 상황

선택한 지역, 연령, 혹은 질병에 대해 충분한 데이터가 없는 경우

### 사용자 기대

왜 결과가 비어 있는지 이해하고, 다음에 무엇을 해야 할지 안내받고 싶다.

### 대응

- 데이터 부족 이유를 간단히 설명한다.
- 다른 연령 또는 인접 지역 확인을 유도할 수 있다.
- 빈 화면처럼 보이지 않도록 최소한의 설명을 제공한다.

## 흐름 B. 관심 지역이 초기 범위 밖인 경우

### 상황

사용자가 전국 단위 지역을 기대하지만, 초기 제품 범위는 서울/경기도에 한정되어 있는 경우

### 사용자 기대

지원 범위를 분명히 이해하고 혼란이 없기를 원한다.

### 대응

- 현재 지원 지역이 서울/경기도임을 분명히 보여준다.
- 추후 확장 예정이라는 점은 말할 수 있지만, 현재 사용 가능한 범위를 우선 명확히 안내한다.

## 흐름 C. 질병을 특정하지 않고 전체 위험만 보는 경우

### 상황

사용자가 질병을 선택하지 않은 채 전체 상태만 훑어보는 경우

### 사용자 기대

무엇이 중요한지 요약을 통해 빠르게 알고 싶다.

### 대응

- 기본 요약 패널에서 현재 주의 질병을 우선 노출한다.
- 필요 시 질병 선택으로 자연스럽게 이어지도록 한다.

## 플로우 설계 원칙

- 메인 플로우는 최대한 짧아야 한다.
- 사용자는 지도 진입 후 곧바로 의미 있는 정보를 얻어야 한다.
- 상세 분석과 예측은 메인 플로우를 방해하지 않고, 필요할 때 깊게 들어가는 구조여야 한다.
- 사용자는 항상 현재 보고 있는 지역, 연령, 질병 상태를 인지할 수 있어야 한다.
- 빈 상태나 범위 제한도 정보 구조의 일부로 설계해야 한다.
