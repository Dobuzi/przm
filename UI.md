# UI.md

## 문서 목적

이 문서는 PRZM의 UI 원칙과 시각적 방향을 정의한다.

PRZM은 공공 통계 리포트 화면이 아니라, 지도 중심으로 위험을 빠르게 읽고 탐색하는 웹 앱이다. 따라서 UI는 많은 정보를 한 번에 나열하기보다, 사용자가 지금 중요한 신호를 빠르게 파악하고 필요한 지점에서만 깊게 들어가도록 설계되어야 한다.

이 문서의 목적은 색상, 레이아웃, 정보 계층, 인터랙션 원칙을 정리해 이후 와이어프레임, 비주얼 디자인, 컴포넌트 설계의 기준으로 사용하는 것이다.

## UI 원칙

- 지도는 배경이 아니라 제품의 중심 인터페이스다.
- 사용자는 몇 초 안에 현재 위험 신호를 읽을 수 있어야 한다.
- 화면은 빠르게 훑히는 구조여야 하며, 긴 설명보다 시각적 상태 전달이 우선이다.
- 복잡한 분석 정보는 처음부터 다 노출하지 않고, 필요할 때 단계적으로 연다.
- 현재 상태, 변화 방향, 예측 정보는 서로 혼동되지 않도록 구분되어야 한다.
- 일반 시민이 사용하는 제품이므로 전문 용어보다 즉시 이해 가능한 표현을 우선한다.

## 시각적 방향

PRZM의 시각적 방향은 "지도 서비스처럼 빠르고 실용적인 탐색 중심"이다.

즉, 사용자는 앱에 들어왔을 때 문서나 보고서를 읽는 느낌보다, 지금 상황을 빠르게 스캔하고 필요한 정보를 집어드는 느낌을 받아야 한다.

### 지향하는 인상

- 빠름
- 명확함
- 실용적 탐색
- 현재성

### 피해야 하는 인상

- 공공 보고서처럼 무겁고 답답한 화면
- 테이블과 긴 문장 위주의 정적 화면
- 과도하게 장식적이거나 감성적인 비주얼
- 신호보다 UI 자체가 더 눈에 띄는 화면

## 레이아웃 원칙

### 1. 지도 우선 레이아웃

가장 넓은 영역은 항상 지도여야 한다.

요약 정보, 필터, 상세 정보는 모두 지도를 보조하는 구조로 배치한다.

### 2. 상단 제어 최소화

지역, 질병, 연령 선택 같은 제어 요소는 항상 접근 가능해야 하지만, 지도 시야를 과하게 가리면 안 된다.

상단 제어 바는 짧고 가볍게 유지하고, 선택 상태는 명확히 드러나야 한다.

### 3. 정보는 단계적으로 열린다

처음부터 모든 분석 차트와 설명을 한 화면에 쌓지 않는다.

초기 상태에서는 현재 위험과 확산 범위만 보여주고, 상세 분석과 예측은 패널 또는 드로어 안에서 연다.

### 4. 상태 유지 중심 구조

사용자가 지역, 연령, 질병을 선택한 뒤 상세 패널을 열고 닫아도 맥락이 유지되어야 한다.

즉, 화면 전환보다 상태 유지가 더 중요하다.

## 지도 위 인터랙션 원칙

### 1. 지역 선택은 자연스럽고 즉각적이어야 한다

사용자는 지도에서 특정 지역을 누르거나 선택했을 때 곧바로 반응을 느껴야 한다.

선택된 지역은 다른 지역과 분명히 구분되어야 한다.

### 2. 확산 범위는 시각적으로 바로 읽혀야 한다

사용자는 질병을 선택했을 때 확산 영역이 어디인지 지체 없이 파악할 수 있어야 한다.

지도 위 표현은 세밀한 설명보다 우선적으로 읽혀야 한다.

### 3. 상호작용 후 결과는 같은 맥락에서 보여야 한다

연령을 바꾸거나 질병을 선택했을 때, 사용자는 다른 화면으로 이동하지 않고 현재 지도에서 결과가 달라지는 것을 봐야 한다.

### 4. 상세 정보는 지도 탐색을 끊지 않아야 한다

상세 패널 또는 드로어가 열려도 사용자는 여전히 지도를 기준점으로 삼을 수 있어야 한다.

## 위험도 시각화 원칙

PRZM의 위험도는 직관적인 색상 단계 중심으로 표현한다.

위험도 색상은 장식 요소가 아니라 핵심 정보 전달 도구다.

### 1. 색상 단계는 즉시 이해 가능해야 한다

사용자는 범례를 오래 읽지 않아도 색을 보고 대략적인 위험 수준을 파악할 수 있어야 한다.

예를 들어 낮은 위험, 중간 위험, 높은 위험이 자연스럽게 구분되어야 한다.

### 2. 현재 위험과 예측 신호는 구분되어야 한다

현재 상태의 위험 색상과 예측 방향 표현이 뒤섞이면 안 된다.

현재 위험은 지도 위 색상 단계로 보여주고, 예측은 상세 레이어 안에서 별도 해석 단위로 보여주는 편이 적절하다.

### 3. 색상은 과도하게 많지 않아야 한다

위험도 단계가 너무 많으면 일반 사용자는 차이를 해석하기 어렵다.

초기 MVP에서는 소수의 명확한 단계가 적합하다.

### 4. 색상만으로 모든 의미를 전달하지 않는다

위험도는 색상이 중심이지만, 선택 상태나 예측 해석은 텍스트와 구조로 함께 보완해야 한다.

## 타이포와 정보 계층

### 1. 핵심 문장은 짧고 강해야 한다

요약 정보와 예측 문장은 짧고 즉시 읽히는 문장이어야 한다.

예를 들어 "현재 위험 높음", "일주일 내 증가 가능성", "선택 연령 기준 주의 필요" 같은 식의 문장이 적합하다.

### 2. 숫자보다 해석이 먼저 보이게 한다

PRZM은 분석 도구이지만, 일반 시민 대상 제품이다.

따라서 숫자와 차트는 보조 정보이며, 사용자가 먼저 보는 것은 해석 문장과 상태 레이블이어야 한다.

### 3. 정보 계층은 현재 → 분석 → 예측 순서다

사용자가 먼저 봐야 할 것은 현재 위험이다.

그 다음이 왜 그런지 설명하는 분석이며, 마지막이 앞으로 어떻게 될지 보여주는 예측이다.

이 순서를 UI에서도 유지해야 한다.

### 4. 선택 상태는 항상 눈에 보여야 한다

사용자는 자신이 지금 어떤 지역, 어떤 연령, 어떤 질병 기준으로 보고 있는지 항상 알아야 한다.

선택 상태는 보조 정보가 아니라 전체 해석의 기준이다.

## 요약 패널 원칙

요약 패널은 사용자가 지도만으로 놓칠 수 있는 핵심 메시지를 빠르게 전달하는 영역이다.

### 역할

- 지금 가장 중요한 위험 신호 전달
- 현재 선택 상태 해석
- 다음 행동 유도

### 피해야 할 점

- 긴 문단 설명
- 지나치게 많은 수치
- 차트만 있고 해석 문장이 없는 구성

요약 패널은 "현재 무엇을 주의해야 하는가"를 먼저 말해야 한다.

## 상세 패널 / 드로어 원칙

상세 정보는 메인 지도에서 더 깊이 들어갈 때 열리는 2차 정보 레이어다.

### 역할

- 추세 설명
- 확산 속도 설명
- 연령/성별 분포 설명
- 일주일/한달 예측 제공

### 원칙

- 현재 상태의 연장선으로 보이도록 구성한다.
- 지도와 완전히 분리된 독립 화면처럼 느껴지면 안 된다.
- 정보는 위에서 아래로 자연스럽게 읽혀야 한다.
- 현재 상태와 예측 상태를 한 맥락 안에서 연결해야 한다.

## 데스크톱 적용 원칙

- 지도는 넓게 유지한다.
- 상세 정보는 오른쪽 패널로 연다.
- 요약과 상세 정보가 동시에 보이더라도 지도가 여전히 중심이어야 한다.
- 사용자는 지도와 분석 내용을 나란히 보며 해석할 수 있어야 한다.

## 모바일 적용 원칙

- 지도 가시성을 최대한 보존한다.
- 요약 정보는 하단 카드 형태로 가볍게 시작한다.
- 상세 정보는 하단 드로어로 확장한다.
- 상단 제어 요소는 최소한의 공간만 차지해야 한다.
- 작은 화면에서도 지역, 연령, 질병 선택 상태를 잃지 않도록 한다.

## 마이크로카피 원칙

- 짧고 직설적인 문장을 쓴다.
- 전문 역학 용어는 피하거나 최소화한다.
- 사용자에게 판단 힌트를 주는 문장을 우선한다.
- 단정적 표현보다 상황 해석 표현이 적합하다.

예시:

- "현재 주의 필요"
- "최근 확산세가 커지는 중"
- "선택 연령에서 위험이 더 높게 나타남"
- "앞으로 일주일간 증가 가능성"

## 초기 범위 기준 UI

초기 UI는 아래 범위에 맞춰 설계한다.

- 지역: 서울특별시, 경기도
- 연령: 0세부터 13세까지
- 사용자: 학부모, 일반 시민
- 핵심 경험: 우리 동네 위험 확인 → 내 아이 연령대 확인 → 상세 분석 → 예측 확인

즉, 전문가용 조밀한 분석 도구보다는 빠르게 읽히는 지도 탐색 제품에 가까워야 한다.

## 현재 단계의 비목표

- 복잡한 대시보드형 위젯 나열
- 다수 차트가 동시에 펼쳐진 분석 화면
- 장식적인 브랜딩 중심 UI
- 문서형 리포트 화면
- 설정 메뉴가 중심이 되는 구조
